This page walks you through getting a local CulturePass environment running end-to-end: install dependencies, deploy a personal AWS Amplify sandbox, wire the generated outputs into your `.env`, and launch the app.

## Prerequisites

Install and configure these before you start:

- **Node.js** 20 or newer.
- **npm** (ships with Node).
- **AWS CLI**, signed in with a profile that can deploy CloudFormation, Cognito, AppSync, DynamoDB, S3, and Lambda. The backend defaults to the `ap-southeast-2` region.
- **AWS Amplify Gen 2 CLI** — invoked as `npx ampx` from the repo, no global install needed.
- **Expo tooling** for the platform you want to target:
  - Web works with just a modern browser.
  - iOS requires macOS with Xcode installed.
  - Android requires Android Studio with an emulator (or a physical device with USB debugging).

## 1. Install dependencies

From the repo root:

```bash
npm install
```

## 2. Deploy a personal Amplify sandbox

Each contributor deploys their own isolated cloud sandbox — Cognito user pool, AppSync API, DynamoDB tables, S3 bucket, and Lambdas — using Amplify Gen 2:

```bash
AWS_PROFILE=your-aws-profile npx ampx sandbox
```

The first run provisions the stack and takes several minutes. Leave the command running; it watches the `amplify/` directory and redeploys on save. When the stack is up, it writes `amplify_outputs.json` to the repo root with the IDs, endpoints, and bucket name the client needs.

If checkout flows are in scope for what you're working on, set the Stripe secrets on the sandbox (they're never exposed to the client bundle):

```bash
npx ampx sandbox secret set STRIPE_SECRET_KEY
npx ampx sandbox secret set STRIPE_WEBHOOK_SECRET
```

## 3. Configure environment

Copy the example file and populate the AWS values from `amplify_outputs.json`:

```bash
cp .env.example .env
node scripts/aws-env-from-outputs.mjs
```

`scripts/aws-env-from-outputs.mjs` reads `amplify_outputs.json` and fills in the `EXPO_PUBLIC_*` variables in `.env` — region, Cognito user pool and app client IDs, Cognito identity pool ID, AppSync GraphQL endpoint, and the S3 bucket name. The backend selector is fixed:

```env
EXPO_PUBLIC_BACKEND=aws
```

Add an optional `EXPO_PUBLIC_MAPBOX_TOKEN` if you're working on map surfaces.

## 4. Seed test data (optional)

The `dev-seed` Lambda creates a buyer Profile plus a published paid Event so the Stripe purchase flow can be exercised end-to-end. Sign up an account in the app first to get a Cognito sub, then invoke the function:

```bash
aws lambda invoke \
  --function-name <dev-seed-function-name> \
  --payload '{"buyerSub":"<your-cognito-sub>"}' \
  out.json
```

The deployed function name is printed by `npx ampx sandbox` and also appears in `amplify_outputs.json`.

The `post-confirmation` Lambda automatically creates a Profile for every newly confirmed Cognito user, so most local sign-ups don't need any extra seeding.

## 5. Run the app

Pick a target platform:

```bash
npm run web        # browser
npm run ios        # iOS simulator (macOS only)
npm run android    # Android emulator or connected device
```

`npm run web` is the fastest feedback loop for most UI work.

## Historical: Migrating an existing dataset (one-time cutover)

**The migration from Supabase to AWS is complete.** The production build is AWS-only (`lib/backend.ts` pins `BACKEND = "aws"`).

The one-time cutover script `scripts/migrate-supabase-to-dynamo.mjs` was used to port data (reference → profiles → hubs → events → tickets → messages) through AppSync. It is no longer part of normal development or onboarding.

You can still run it for reference if you have an old Supabase dump:

```bash
node scripts/migrate-supabase-to-dynamo.mjs
# preview:
DRY_RUN=1 node scripts/migrate-supabase-to-dynamo.mjs
```

## Useful scripts

| Script | What it does |
| --- | --- |
| `npm run web` / `ios` / `android` | Start the Expo app on the chosen platform. |
| `npm run lint` | Run ESLint across the codebase. |
| `npm run typecheck` | Run `tsc --noEmit`. |
| `npx ampx sandbox` | Deploy and watch your personal Amplify backend. |
| `node scripts/aws-env-from-outputs.mjs` | Populate `.env` from `amplify_outputs.json`. |
| `npm run migrate:dynamo` | Run the one-shot Supabase → DynamoDB migration. |
| `npm run migrate:dynamo:dry` | Dry-run the migration without writing. |
